Monkey Music Challenge is a programming challenge for Chalmers and GU students who love to code. The challenge consists of constructing your own AI. The AI's main tasks are pathfinding and simple decision making.

The event(s)

The challenge is split into two separate events. Wait what? Yeh, we figured not all of us crank out our best code during hackathons. We believe many of us work better after having a few nights of digesting a problem!

So we'll have two different events. The first event will introduce the challenge and the engine that you will communicate with during the challenge. The second event will be split into two parts, a morning where you get to finish your AI at our office. And an afternoon with presentations AI battles!

We'd like you to apply in teams of two, since programming together is fun. We'll have 20 team spots available. We go by "first come-first serve" for the signup, but those who take the Warmup test will get ahead in line. Read more under "Getting Started" below.

Getting started

So you've signed up – now what? You can and should start preparing by taking the warm up test to test your skills and get a hang of the concept. Those who finish the warmup exercise will also get ahead in line when we're picking out the teams for the contest.

  1. Take the warmup test

    The warmup is kind of a single player version of the real version. By doing the warmup well, you'll be prepared for the real deal later! We'll store your team names when (or if?) you finish the warmup.

  2. Start studying the starter code

    We’ve created startkits for the most common languages. These starters will help you get up and running writing the actual logic instead of going through the initial hassle of setting up the communication with our server with POST requests.

    • .js
    • .py
    • .java

Tech info

The short story: you'll control a monkey with the goal of collecting music on a tiled map and delivering it to a human. For this you need to write pathfinding algorithms. You'll compete with the other teams' monkeys with this algorithm.

At a glance

  • We have a server who has the map representation.
  • You write the logic for monkey movement.
  • You'll control the monkey by sending POST requests to this server with instructions on how the monkeys should move.
  • You're free to write the logic in any language you'd like, as long as it's capable of sending POST requests (we have starter code for some popular languages in our repo).

API endpoints

An API key is required in all requests (attached in the apiKey attribute). Request data should be JSON (application/json).

New game

Command
new game
Endpoint
/<your-team-name>
Sample
var data = {
  command: 'new game',
  apiKey: 'your-key'
};

client.post('/your-team-name', data, callback);

Movement

Command
move
Endpoint
/<your-team-name>
Required keys
direction String being one of up|left|down|left
Sample
var direction = move();   // Actual game logic
var data = {
  command: 'move',
  direction: direction,
  apiKey: 'your-key'
};

client.post('/your-team-name', data, callback);

Server response

When doing your POST request, you'll receive a JSON response from our server which will come handy for calculating your next step. The response looks as follows: 

{
  layout: Array,
  turns: Number,
  pickedUpItems: Array
}

layout is a matrix describing the current state of the map. The elements are strings describing a tile, for example "monkey", "song" or "album". pickedUpItems is an array of strings describing, yeah, the items you've picked up.
